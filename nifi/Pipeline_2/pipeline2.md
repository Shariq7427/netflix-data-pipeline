# Pipeline 2: Data Processing using Apache NiFi

## Overview

Pipeline 2 is the second stage of the Netflix Data Engineering Project. After the raw dataset is ingested in Pipeline 1, this pipeline processes the CSV data and prepares it for loading into the PostgreSQL data warehouse. It acts as the transformation layer of the ETL workflow.

---

# Objective

The objective of Pipeline 2 is to process the Netflix CSV dataset, validate the records, and prepare clean data for loading into the PostgreSQL database.

---

# Technologies Used

- Apache NiFi 1.28.1
- CSV Dataset
- PostgreSQL
- Windows Operating System

---

# Input

Source:

```
Pipeline 1 Output
```

Input Format:

- CSV File

---

# Apache NiFi Components Used

## GetFile

Purpose:
Reads the processed CSV file generated by Pipeline 1.

Configuration:
- Input Directory
- Keep Source File = True

---

## ConvertRecord

Purpose:
Reads CSV records and converts them into structured records that can be processed by Apache NiFi.

Configuration:
- CSVReader
- CSVRecordSetWriter

---

## PutFile

Purpose:
Stores the processed dataset in the Pipeline 2 output folder for the next ETL stage.

---

# Workflow

```
Pipeline 1 Output
        │
        ▼
     GetFile
        │
        ▼
  ConvertRecord
        │
        ▼
     PutFile
```

# Data Flow

1. Apache NiFi reads the CSV file produced by Pipeline 1.
2. ConvertRecord parses the CSV records into structured data.
3. The processed records are validated and prepared for database loading.
4. The resulting dataset is written to the Pipeline 2 output directory.
5. Pipeline 3 consumes this processed dataset for loading into PostgreSQL.

---

# Output

- Cleaned and processed Netflix dataset.
- Structured CSV file ready for database loading.

---

# Business Use Case

Before loading data into a database, organizations often need to standardize and validate incoming datasets. Pipeline 2 automates this preparation process, ensuring that downstream systems receive consistent and well-structured data.
