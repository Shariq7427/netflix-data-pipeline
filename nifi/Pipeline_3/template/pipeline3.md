# Pipeline 3: Loading Netflix Data into PostgreSQL

## Overview

Pipeline 3 is the database loading stage of the Netflix Data Engineering Project. After the dataset has been ingested and processed, this pipeline loads the Netflix records into a PostgreSQL data warehouse. It demonstrates how Apache NiFi integrates with relational databases using JDBC and connection pooling.

---

# Objective

The objective of Pipeline 3 is to automate the loading of the processed Netflix dataset into a PostgreSQL database, creating a centralized data warehouse for analytics and reporting.

---

# Technologies Used

- Apache NiFi 1.28.1
- PostgreSQL
- PostgreSQL JDBC Driver

# Apache NiFi Components Used

## GetFile

Purpose:
Reads the processed CSV file generated by Pipeline 2.

Configuration:
- Input Directory
- Keep Source File = True

---

## ConvertRecord

Purpose:
Reads CSV data and converts each row into structured records suitable for database insertion.

Controller Services:
- CSVReader
- JsonRecordSetWriter (or Record Writer used during implementation)

---

## PutDatabaseRecord

Purpose:
Loads structured records into the PostgreSQL table.

Configuration:
- Database Connection Pool
- Table Name = netflix_titles

---

## DBCPConnectionPool

Purpose:
Creates a reusable JDBC connection to PostgreSQL.

Configuration:

- Database URL

```
jdbc:postgresql://localhost:5432/netflix_dw
```

- Database

```
netflix_dw
```

- JDBC Driver

```
postgresql-42.7.12.jar
```

---

# Workflow

```
Pipeline 2 Output
        │
        ▼
     GetFile
        │
        ▼
  ConvertRecord
        │
        ▼
 PutDatabaseRecord
        │
        ▼
 PostgreSQL
```

---

# Database Verification

After execution, the following SQL query was used to verify successful loading:

```sql
SELECT COUNT(*) FROM netflix_titles;
```

Expected Result:

```
8807 Records
```

This confirmed that the entire Netflix dataset was successfully loaded into PostgreSQL.

---

# Data Flow

1. Apache NiFi reads the processed CSV file.
2. CSV records are converted into structured records.
3. DBCPConnectionPool establishes a connection to PostgreSQL.
4. PutDatabaseRecord inserts the records into the `netflix_titles` table.
5. PostgreSQL stores the data for downstream analytics.

---

# Output

- Netflix dataset successfully stored in PostgreSQL.
- Approximately **8,807 records** loaded into the `netflix_titles` table.
- Database ready for Power BI reporting and further ETL processing.

# Business Use Case

Modern organizations store processed data inside centralized databases rather than flat files. Pipeline 3 demonstrates how Apache NiFi can automate database loading, reducing manual effort and ensuring consistent, reliable data availability for analytics.
