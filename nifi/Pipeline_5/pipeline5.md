# Pipeline 5: JSON to CSV Transformation using Apache NiFi

## Overview

Pipeline 5 is the data transformation stage of the Netflix Data Engineering Project. This pipeline reads the JSON file generated in Pipeline 4 and converts it into CSV format. It demonstrates Apache NiFi's record-oriented processing capabilities and showcases how data can be transformed between different formats without modifying the underlying data.

---

# Objective

The objective of Pipeline 5 is to transform the Netflix dataset from JSON format into CSV format for downstream processing and reporting.

---

# Technologies Used

- Apache NiFi 1.28.1
- JSON
- CSV
- JsonTreeReader
- CSVRecordSetWriter

---

# Input

Source:

```
Pipeline 4 Output
```

Input Format:

```
JSON
```

---

# Apache NiFi Components Used

## GetFile

Purpose:

Reads the JSON file generated by Pipeline 4.

Configuration:

- Input Directory
- Keep Source File = True

---

## ConvertRecord

Purpose:

Converts JSON records into CSV records.

Configuration:

Record Reader

```
JsonTreeReader
```

Record Writer

```
CSVRecordSetWriter
```

---

## JsonTreeReader

Purpose:

Reads JSON records and converts them into Apache NiFi record objects.

Configuration:

- Schema Access Strategy = Infer Schema

---

## CSVRecordSetWriter

Purpose:

Writes Apache NiFi records into CSV format.

Configuration:

- Default CSV configuration
- Comma-separated output

---

## PutFile

Purpose:

Stores the generated CSV file inside the Pipeline 5 output directory.

---

---

# Data Flow

1. GetFile monitors the Pipeline 4 output directory.
2. The JSON dataset is read into Apache NiFi.
3. JsonTreeReader parses the JSON records.
4. ConvertRecord transforms the JSON records into CSV records.
5. CSVRecordSetWriter generates the CSV file.
6. PutFile stores the converted CSV file in the Pipeline 5 output directory.
7. Pipeline 6 uses this CSV file for further processing.

---

# Output

- Successfully converted the Netflix dataset from JSON to CSV.
- Output file stored inside the Pipeline 5 output directory.

# Business Use Case

Organizations frequently exchange data in multiple formats depending on system requirements. Some applications consume JSON, while others require CSV files. Pipeline 5 demonstrates how Apache NiFi can automate these format conversions without requiring custom programming.
